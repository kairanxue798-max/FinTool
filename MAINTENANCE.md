## 📋 日常维护需求

### 🟢 基本不需要维护（自动运行）

你的应用部署后，大部分情况下**不需要手动维护**：

1. **自动部署**
   - 当你推送代码到 GitHub `main` 分支时
   - Vercel 和 Render 会自动检测并重新部署
   - 无需手动操作

2. **自动运行**
   - 服务会自动保持运行
   - 无需手动启动或重启

3. **自动更新**
   - 依赖更新会自动处理
   - 平台会自动处理安全更新

## ⚠️ 需要注意的事项

### 1. Render 免费版限制（后端）

**重要**：Render 免费版在 30 分钟无活动后会休眠

- **影响**：首次访问可能需要等待 50 秒左右唤醒
- **解决方法**：
  - 这是正常的，等待即可
  - 或者升级到付费版（如果访问量大）

### 2. 监控和检查

**建议定期检查**（每月一次即可）：

1. **检查服务状态**
   - Render Dashboard: https://dashboard.render.com
   - Vercel Dashboard: https://vercel.com/dashboard
   - 确认服务状态为 "Live" 或 "Ready"

2. **检查错误日志**
   - 如果用户报告问题，查看日志
   - Render 和 Vercel 都提供详细的日志

3. **测试功能**
   - 偶尔访问应用测试功能
   - 确保所有功能正常工作

### 3. 代码更新

**当你需要更新代码时**：

1. **本地修改代码**
2. **提交并推送**：
   ```bash
   git add .
   git commit -m "你的更新说明"
   git push origin main
   ```
3. **自动部署**：Vercel 和 Render 会自动重新部署
4. **等待完成**：通常 3-5 分钟

## 🆘 常见问题和处理

### 问题 1: 后端响应慢或超时

**原因**：Render 免费版休眠

**处理**：
- 等待 50 秒左右
- 或者访问后端健康检查：`https://fin-tool-backend-nsl4.onrender.com/api/health`
- 这会唤醒服务

### 问题 2: 部署失败

**处理**：
1. 查看部署日志
2. 检查错误信息
3. 修复代码问题
4. 重新推送代码

### 问题 3: 功能不工作

**处理**：
1. 检查浏览器控制台（F12）
2. 查看 Render 和 Vercel 日志
3. 确认环境变量设置正确
4. 测试后端 API 是否正常
